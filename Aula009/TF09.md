# TF09 - Deploy de Aplicação em EC2 com Segurança de Rede

**Disciplina:** Implementação de Sistemas  
**Curso:** Análise e Desenvolvimento de Sistemas - UniFAAT  
**Valor:** 2,0 pontos  
**Tipo:** Trabalho prático individual  
**Prazo:** Conforme cronograma da disciplina

## Objetivo

Demonstrar competência na criação de infraestrutura AWS usando EC2, implementando uma arquitetura segura com VPC, Security Groups e boas práticas de segurança de rede.

## Descrição da Atividade

Você deve criar uma infraestrutura completa na AWS para hospedar uma aplicação web, implementando todos os conceitos de segurança de rede e computação em nuvem estudados na aula.

## Cenário do Projeto

### Sistema de Portfólio Pessoal
Crie um sistema de portfólio pessoal que demonstre suas habilidades técnicas, contendo:
- **Frontend**: Página web responsiva com suas informações
- **Backend**: API para gerenciar projetos e experiências
- **Database**: Armazenamento de dados dos projetos
- **Infraestrutura**: EC2 com segurança de rede implementada

## Especificações Técnicas

### Instâncias EC2
- **Tipo recomendado**: t3.micro (Free Tier)
- **AMI**: Amazon Linux 2 ou Ubuntu 20.04 LTS
- **Região**: us-east-1 (para aproveitar Free Tier)

### Aplicação
- **Frontend**: HTML/CSS/JavaScript ou framework de sua escolha
- **Backend**: Node.js, Python Flask/Django, ou Java Spring Boot
- **Database**: MySQL, PostgreSQL, ou SQLite para desenvolvimento
- **Containerização**: Docker recomendado mas não obrigatório

### Segurança
- **SSH**: Apenas do seu IP público
- **HTTP/HTTPS**: Acesso público para a aplicação
- **Database**: Acesso apenas do web server
- **Logs**: Configuração adequada de logging

## Dicas para Sucesso

### Planejamento
1. **Desenhe a arquitetura** antes de começar a implementar
2. **Defina os CIDRs** cuidadosamente para evitar conflitos
3. **Liste todas as portas** necessárias antes de criar Security Groups
4. **Planeje a estrutura** de pastas e documentação

### Implementação
1. **Teste incrementalmente** - não deixe tudo para o final
2. **Use scripts** para automatizar criação de infraestrutura
3. **Documente conforme desenvolve** - não deixe para depois
4. **Faça backup** das configurações importantes

### Segurança
1. **Aplique menor privilégio** em todas as regras
2. **Restrinja SSH** ao seu IP específico
3. **Use HTTPS** sempre que possível
4. **Monitore logs** regularmente
