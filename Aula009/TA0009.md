## 1. Conceitos Fundamentais

### Amazon EC2: Elastic Compute Cloud

**Amazon EC2** é o serviço de computação em nuvem da AWS que fornece capacidade de computação redimensionável. É como alugar computadores virtuais que você pode configurar e usar conforme necessário, pagando apenas pelo tempo de uso.

#### Analogia: EC2 como Aluguel de Carros
- **Tipos de instância** = Diferentes modelos de carro (econômico, SUV, esportivo)
- **AMI** = Configuração inicial do carro (combustível, acessórios)
- **Key Pairs** = Chaves do carro
- **Security Groups** = Sistema de segurança e alarme

### Tipos de Instâncias EC2

#### Famílias de Instâncias:

##### 1. **General Purpose (Uso Geral)**
- **t3, t4g**: Burstable performance, ideal para desenvolvimento
- **m5, m6i**: Balanceadas, boas para aplicações web
- **a1**: Baseadas em ARM, custo-efetivas

##### 2. **Compute Optimized (Otimizadas para Computação)**
- **c5, c6i**: Alto desempenho de CPU
- **c6g**: Baseadas em ARM Graviton2

##### 3. **Memory Optimized (Otimizadas para Memória)**
- **r5, r6i**: Muita RAM, ideais para bancos de dados
- **x1e**: Memória extrema para SAP HANA

##### 4. **Storage Optimized (Otimizadas para Armazenamento)**
- **i3**: NVMe SSD, alta IOPS
- **d3**: HDD denso para data warehousing

##### 5. **Accelerated Computing (Computação Acelerada)**
- **p4**: GPUs para machine learning
- **g4**: GPUs para workstations gráficas

### AMI: Amazon Machine Images

**AMI** é um template que contém configuração de software necessária para lançar uma instância. É como uma "foto" de um sistema operacional configurado.

#### Componentes de uma AMI:
- **Root volume template**: Sistema operacional base
- **Launch permissions**: Quem pode usar a AMI
- **Block device mapping**: Configuração de armazenamento

#### Tipos de AMI:
- **Amazon Linux**: Otimizada para AWS
- **Ubuntu**: Popular para desenvolvimento
- **Windows Server**: Para aplicações Microsoft
- **Red Hat**: Para ambientes empresariais

### Security Groups: Firewall Virtual

**Security Groups** funcionam como firewall virtual para instâncias EC2, controlando tráfego de entrada e saída.

#### Características:
- **Stateful**: Se você permite entrada, a saída é automaticamente permitida
- **Deny by default**: Tudo é bloqueado por padrão
- **Allow rules only**: Apenas regras de permissão, não de negação

#### Exemplo de Configuração:
```
Inbound Rules:
- HTTP (80) from 0.0.0.0/0
- HTTPS (443) from 0.0.0.0/0  
- SSH (22) from 203.0.113.0/24

Outbound Rules:
- All traffic to 0.0.0.0/0
```

### Chaves SSH: Acesso Seguro

**SSH Key Pairs** são usadas para acesso seguro às instâncias Linux. Consistem em:
- **Private Key**: Mantida secreta pelo usuário
- **Public Key**: Armazenada na instância EC2

#### Processo de Conexão:
1. AWS armazena a chave pública na instância
2. Usuário usa chave privada para conectar
3. Autenticação criptográfica sem senhas

### VPC: Virtual Private Cloud

**VPC** é uma rede virtual isolada na AWS onde você pode lançar recursos. É como ter sua própria rede privada na nuvem.

#### Componentes da VPC:

##### 1. **Subnets (Sub-redes)**
- **Public Subnet**: Tem acesso direto à internet
- **Private Subnet**: Sem acesso direto à internet

##### 2. **Internet Gateway**
Permite comunicação entre VPC e internet

##### 3. **Route Tables**
Definem para onde o tráfego de rede é direcionado

##### 4. **NAT Gateway**
Permite que instâncias em subnets privadas acessem a internet

### Armazenamento EC2

#### EBS: Elastic Block Store
- **Persistente**: Dados sobrevivem ao término da instância
- **Tipos**: gp3 (SSD geral), io2 (alta IOPS), st1 (throughput)
- **Snapshots**: Backups incrementais

#### Instance Store
- **Temporário**: Dados perdidos quando instância para
- **Alta performance**: Acesso direto ao hardware
- **Sem custo adicional**: Incluído no preço da instância

### Modelos de Precificação

#### 1. **On-Demand**
- Pague por hora/segundo de uso
- Sem compromissos
- Ideal para workloads imprevisíveis

#### 2. **Reserved Instances**
- Desconto de até 75%
- Compromisso de 1-3 anos
- Ideal para uso constante

#### 3. **Spot Instances**
- Desconto de até 90%
- Podem ser interrompidas
- Ideal para workloads flexíveis

#### 4. **Dedicated Hosts**
- Servidor físico dedicado
- Compliance e licenciamento
- Custo mais alto

## 2. Questões de Múltipla Escolha

### Questão 1
Qual é a principal característica dos Security Groups na AWS?

a) São stateless e requerem regras de entrada e saída separadas
b) São stateful e permitem tráfego de retorno automaticamente
c) Funcionam apenas no nível de subnet, não de instância
d) Permitem apenas regras de negação (deny), não de permissão

**Gabarito: B**

**Justificativa:** Security Groups são stateful, o que significa que se você permitir tráfego de entrada, o tráfego de resposta correspondente é automaticamente permitido na saída, sem necessidade de regra explícita. Isso os diferencia de NACLs que são stateless. As outras opções estão incorretas.

### Questão 2
Qual tipo de instância EC2 é mais adequado para aplicações que requerem muito processamento de CPU?

a) Memory Optimized (r5, r6i)
b) Storage Optimized (i3, d3)
c) Compute Optimized (c5, c6i)
d) General Purpose (t3, m5)

**Gabarito: C**

**Justificativa:** Instâncias Compute Optimized (como c5, c6i) são especificamente projetadas para workloads que requerem alto desempenho de CPU, como computação científica, modelagem, análise de dados e servidores web de alta performance. As outras famílias são otimizadas para diferentes recursos.

### Questão 3
O que acontece com os dados armazenados no Instance Store quando uma instância EC2 é parada?

a) Os dados são automaticamente transferidos para o EBS
b) Os dados são mantidos e ficam disponíveis quando a instância reinicia
c) Os dados são perdidos permanentemente
d) Os dados são salvos em um snapshot automático

**Gabarito: C**

**Justificativa:** Instance Store é armazenamento temporário que está fisicamente conectado ao servidor host. Quando uma instância é parada (stopped) ou terminada, todos os dados no Instance Store são perdidos permanentemente. Apenas dados em volumes EBS persistem após parar a instância.
